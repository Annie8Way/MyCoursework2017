=== Markdown

Markdown считается более лёгким языком разметки, но это и является его главным недостатком:

* Markdown изначально может конвертировать только в HTML. Для конвертации в другие форматы требуются дополнительные расширения.
* Так же "чистый" Markdown не содержит синтаксиса для написания таблиц. Опять же для упрощения написания придётся установить плагины.

*AsciiDoc уже включает в себя все нужные инструменты!*

== Распределённые системы контроля версий

image::skv.png[skv,100%,float="left"]

*Распределённая система контроля версий* - система, регистрирующая изменения в одном или нескольких файлах с тем, чтобы в дальнейшем была возможность вернуться к определённым старым версиям этих файлов. В данной системе клиенты не просто выгружают последние версии файлов с сервера, а полностью копируют весь репозиторий.

=== Git и Mercurial

Основные команды практически одинаковы. Пример на Git:

* `git clone git://github.com/project` - клонирует репозиторий на локальный компьютер
* `git status` - показывает, какие файлы изменены, но не добавлены в индекс
* `git diff` - показывает конкретную разницу между вашей рабочей директорией и индексом
* `git checkout <хэш коммита или тег>` - переход на определённую версию
* `git add` - добавляет изменённый файл в индекс (staging area)
* `git commit -m "комментарий"`
* `git push origin` - проталкивает коммит на удалённый репозиторий
* `git pull` - смешивает изменения на удалённом репозитории с вашим локальным

=== Зачем же тогда Telegram? image:telegram.png[telegram,10%]

Telegram - не просто чат, а *площадка интересов*, где каждый может создать глобальный канал на определённую тему, например: разработка приложений, разработка на Python, веб-дизайн, новости IT и пр.

*Telegram предоставляет возможности:*

* быстроого поиска единомышленников для проекта
* поиска решения проблем при какой-либо разработке
* интеграции со сторонними приложениями (тот же GitHub)

== Способы автоматизации процесса обучения

* Написать скрипт, который будет проверять почту, скачивать работы учащихся, сверять их с правильным решением и записывать результаты в отдельный файл
* Создать сайт, разместить на нём тесты и задачи для самопроверки, доступ к которым открывается по ходу учебного процесса
* Разместить на сайте учебные и информационные материалы, обеспечив к ним онлайн доступ

== Выводы

* В качестве языка разметки, исходя из начального функционала и форматов конвертации, нужно использовать AsciiDoc.
* Редактировать текст с разметкой AsciiDoc лучше всего в Sublime Text, так как спектр его инструментов и плагинов является наиболее широким среди аналогов.
* Чтобы избежать потерь информации, иметь возможность управлять этапами проекта, нужно использовать распределённые системы контроля версий Git или Mercurial.

=== Выводы

* Slack с интеграцией GitHub наиболее подходит для поддержания связи с коллегами и учащимися в процесе создания проекта/обучения. Telegram полезен для поиска единомышленников и решения различных проблем разработки.
* Автоматизация - безусловно нужная вещь, позволяющая сэкономить время и ресурсы. Необходимо внедрить её в процесс обучения, реализовав приведённые алгоритмы.
