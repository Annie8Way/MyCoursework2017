Использование современных информационных технологий в методическом обеспечении учебного процесса

Цель работы

Изучить наиболее популярные информационные технологии, потенциально применимые в области оформления документов, учебных и информационных материалов, и рассмотреть возможность их внедрения в учебный процесс.

Решаемые задачи

  • Выбрать наиболее удобный язык разметки, поддерживающий конвертацию документа в другие известные форматы (HTML5, PDF, eBook)

  • Сравнить функциональность популярных текстовых редакторов (Sublime, Brackets, Atom) и проанализировать результаты

  • Рассмотреть распределённые системы контроля версий Git и Mercurial

  • Изучить функционал наиболее популярных чатов (Slack и Telegram)

  • Описать возможные способы автоматизации процесса обучения

Актуальность работы

В данной работе будут рассматриваться технологии, которые позволят:

  • Предоставить обучающимся свободный и постоянный доступ к учебным материалам курса, тем самым исключив проблемы с нехваткой бумажных учебных пособий

  • Обеспечить гибкий версионный контроль над разработкой учебного курса с возможностью хранения проекта на удалённом сервере

  • Упростить процесс создания и размещения учебных материалов в сети Интернет

  • Установить контакт обучающегося с преподавателем с помощью специальных чатов

  • Сэкономить время преподавателя, частично автоматизировав проверку домашних заданий

Языки разметки Markdown & AsciiDoc

Язык разметки (англ. markup language) – это термин, используемый для обозначения набора символов и последовательностей, определяющих строение документа и порядок его визуализации.

Markdown

Markdown считается более лёгким языком разметки, но это и является его главным недостатком:

  • Markdown изначально может конвертировать только в HTML. Для конвертации в другие форматы требуются дополнительные расширения.

  • Так же "чистый" Markdown не содержит синтаксиса для написания таблиц. Опять же для упрощения написания придётся установить плагины.

Текстовые редакторы

Самыми популярными кроссплатформенными текстовыми редакторами на данное время являются:

Скорость

По сравнению со своими аналогами, Sublime требует более низкие системные требования и быстрее запускается, в то время как пользователи Brackets пишут о сбоях при работе с особо большими документами.

Простота использования

Большинство функций Brackets можно настроить прямо в меню интерфейса, не залезая в файлы конфигурации. Хоть функционал Sublime и более широк, поиск и конфигурация подходящих настроек занимает время.

Поддержка синтаксиса

Конечно, текст можно форматировать в любом редакторе, но именно Sublime Text предоставляет более широкий выбор подсветки синтаксиса разнообразных языков, нежели Brackets и Atom

Веб-утилиты

Brackets создавался с целью упростить веб-разработку. Он поддерживает:

  • Возможность "вживую" наблюдать процесс изменения документа в браузере (Live Preview)

  • Быстрое редактирование CSS: редактируя HTML зажать Ctrl, чтобы изменить стиль выбранного элемента.

  • Палитра цветов, таймер анимаций, редактор кривых Безье

Плагины

Sublime предоставляет свыше 2500 расширений, однако их качество - это уже совсем другой вопрос. Atom и Brackets располагают меньшим количеством плагинов, но зато хорошо задокументированных.

Распределённые системы контроля версий

Распределённая система контроля версий - система, регистрирующая изменения в одном или нескольких файлах с тем, чтобы в дальнейшем была возможность вернуться к определённым старым версиям этих файлов. В данной системе клиенты не просто выгружают последние версии файлов с сервера, а полностью копируют весь репозиторий.

Чаты для работы/учёбы

Slack - это онлайн-сервис для ведения переписки внутри команды и большой компании, поддерживающий интеграцию разнообразных приложений, например GitHub.

Slack позволяет:

  • Отрегулировать уведомления по упоминанию имени

  • Создавать чат под любую тему

  • Свободно использовать 5 Гб для файлов общего доступа

  • Настроить уведомления об изменениях проекта на GitHub

  • Использовать чат-ботов для составления различной статистики (выполнение поставленных задач, активность участников проекта и т.д.)

Зачем же тогда Telegram?

Telegram - не просто чат, а площадка интересов, где каждый может создать глобальный канал на определённую тему, например: разработка приложений, разработка на Python, веб-дизайн, новости IT и пр.

Telegram предоставляет возможности:

  • быстроого поиска единомышленников для проекта

  • поиска решения проблем при какой-либо разработке

  • интеграции со сторонними приложениями (тот же GitHub)
